## 定义

保证类任何情况下只有一个实例



优点：

在内存只有一个实例，减少乐内存开销

缺点：

没有接口，扩展困难



注意点：

- 私有构造器
- 线程安全
- 延迟加载
- 序列化和反序列化安全
- 反射（防御）



## 代码

**一、懒汉式，加锁保证线程安全**

```java
public class LazySingleton {
    private static LazySingleton lazySingleton = null;

    private LazySingleton(){}

    /**
     * 线程不安全
     */
    public static LazySingleton getInstanceUnSafe(){
        if(lazySingleton == null){
            lazySingleton = new LazySingleton();
        }
        return lazySingleton;
    }

    /**
     * 加锁
     */
    public static synchronized LazySingleton getInstance(){
        if(lazySingleton == null){
            lazySingleton = new LazySingleton();
        }
        return lazySingleton;
    }
}
```

**二、懒汉式，Double Check**

```java
public class LazyDoubleCheckSingleton {
    // volatile 禁止重排序
    private static volatile LazyDoubleCheckSingleton instance = null;

    private LazyDoubleCheckSingleton(){}

    public static LazyDoubleCheckSingleton getInstance(){
        if(instance==null){
            synchronized (LazyDoubleCheckSingleton.class){
                if(instance == null){
                    instance = new LazyDoubleCheckSingleton();
                }
            }
        }
        return instance;
    }
}
```

**三、静态内部类**

```java
public class StaticInnerClassSingleton {
    // 静态内部类
    private static class InnerClass{
        private static StaticInnerClassSingleton instance
                = new StaticInnerClassSingleton();
    }

    // jvm初始化类时会通过锁保证线程安全
    public static StaticInnerClassSingleton getInstance(){
        return InnerClass.instance;
    }
}
```

**四、饿汉式**

```java
public class HungrySingleton {
    private final static HungrySingleton instance = new HungrySingleton();
    private HungrySingleton(){
        
    }
    public static HungrySingleton getInstance(){
        return instance;
    }
}
```

## 破坏单例

- 序列化到文件之后，再反序列化得到的对象和原对象不同

- 通过反射，调用私有构造器

  ```java
  // 解决办法，在私有构造器中添加代码逻辑进行判断
  private HungrySingleton(){
  	if(instance != null){
          throw RuntimeException("单例构造器禁止反射调用");
      }        
  }
  ```
